# FocusPulse MVP - Product Requirements Document

## Overview

**Product Name:** FocusPulse  
**Domain:** focuspulse.app  
**Timeline:** 1 week development  
**Target Launch Date:** Week of November 15, 2025

## Product Vision

FocusPulse helps professionals build a consistent deep work habit through planning, tracking, and streak-based motivation. Unlike existing tools that only track time, FocusPulse emphasizes proactive planning and habit formation.

---

## Core Value Proposition

"Plan your deep work sessions, track your progress, and build an unbreakable streak"

### Key Differentiators:

1. **Planning-first approach** - Schedule deep work blocks in advance
2. **Habit formation focus** - Streak-based motivation
3. **Simplicity** - Clean, focused UI without feature bloat
4. **Multi-device sync** - Access your data across all devices

---

## Target Audience

### Primary Users:

- Knowledge workers (developers, designers, writers, researchers)
- Freelancers who need to manage their own time
- Remote workers struggling with focus
- Students working on long-term projects

### User Pain Points:

- Difficulty maintaining consistent deep work habits
- Reactive time tracking (logging after the fact)
- Feature-bloated productivity tools
- No visibility into when they're most productive
- Lack of accountability and motivation

---

## User Stories

### As a new user:

- I want to quickly sign up and understand the app's value
- I want to schedule my deep work sessions for the week
- I want to start a focus session immediately

### As a returning user:

- I want to see my current streak when I log in
- I want to start my pre-planned deep work session
- I want to log an unplanned deep work session
- I want to see how much deep work I've done this week

### As an engaged user:

- I want to see patterns in my productivity (best days/times)
- I want to adjust my weekly deep work goals
- I want to review my past sessions and notes
- I want to maintain my streak

---

## Core Features (MVP)

### 1. User Authentication

**Must Have:**

- Sign up with email and password
- Sign in with email and password
- Sign out
- Password reset via email
- Email verification (optional for MVP)

**User Flow:**

1. New user lands on homepage
2. Clicks "Sign Up"
3. Enters email, password, full name
4. Receives welcome email
5. Logs in and sees dashboard

---

### 2. Dashboard (Home Page)

**Purpose:** Central hub showing today's activity and motivation metrics

**Must Display:**

- Current streak (prominently at top)
  - Example: "ğŸ”¥ 15 Day Streak"
- Today's progress
  - Completed sessions for today
  - Upcoming planned sessions
- Weekly progress
  - Hours completed this week
  - Progress toward weekly goal (e.g., "8.5 / 12 hours")
- Calendar heatmap (monthly view)
  - Shows which days user completed deep work
  - Color intensity based on duration
- Primary CTA: "Start Deep Work Now" button

**User Actions:**

- Start an ad-hoc deep work session
- View today's completed sessions
- Click on planned sessions to start them
- Navigate to other pages

---

### 3. Weekly Planner

**Purpose:** Proactively schedule deep work blocks for the week

**Must Have:**

- Weekly calendar grid (Monday - Sunday)
- Time slots (8 AM - 8 PM minimum)
- Ability to add deep work blocks
  - Click and drag to create blocks
  - Or click to add with form (time, duration, title)
- Each block shows:
  - Time (e.g., "9:00 - 10:30 AM")
  - Title (e.g., "Write report")
  - Duration indicator
- Edit existing blocks
- Delete blocks
- Blocks persist across devices

**User Flow:**

1. User navigates to "Plan" page
2. Sees current week's calendar
3. Clicks/drags to add a deep work block
4. Enters details (title, time, duration)
5. Block appears on calendar
6. Can rearrange or delete as needed

**Visual Design:**

- Clear time labels on Y-axis
- Days of week on X-axis
- Color-coded blocks
- Responsive for mobile (stacked view)

---

### 4. Active Session Timer

**Purpose:** Track an active deep work session in real-time

**Must Have:**

- Full-screen or focused view (minimal distractions)
- Running timer display (counts up)
  - Example: "01:23:45"
- Display what user is working on
- "End Session" button
- Optional: "Take Break" button (pauses timer)

**Starting a Session:**

- From planned block: Click "Start" on scheduled session
- Ad-hoc: Click "Start Deep Work Now" from dashboard
- When starting, user enters:
  - What they're working on (title)
  - Expected duration (optional)

**Ending a Session:**

- User clicks "End Session"
- Shows completion modal:
  - Actual duration
  - Option to add notes about what was accomplished
  - "Save Session" button

**User Flow:**

1. User clicks to start session
2. Enters title (e.g., "Design mockups")
3. Timer starts
4. User works in full-screen mode
5. Clicks "End Session" when done
6. Adds notes if desired
7. Session saved to history

---

### 5. Session History & Stats

**Purpose:** Review past work and identify productivity patterns

**Must Have:**

**History View:**

- List of all past sessions (newest first)
- Each entry shows:
  - Date
  - Time range
  - Duration
  - Title
  - Notes (if any)
  - Whether it was planned or ad-hoc
- Filter options:
  - By date range (this week, last week, this month)
  - By completed/planned status
- Search by title/notes

**Stats Dashboard:**

- Total deep work hours (all time, this month, this week)
- Current streak
- Longest streak ever
- Total number of sessions
- Average session duration
- Charts/graphs:
  - Hours per week (bar chart)
  - Hours per day of week (to identify best days)
  - Monthly trend

**Insights Section (Nice to Have):**

- "Your most productive day is Tuesday"
- "You're most consistent in the mornings"
- "Your average session length is 85 minutes"

---

### 6. Settings & Profile

**Must Have:**

- Edit profile information
  - Full name
  - Email (with verification if changed)
- Change password
- Set weekly goal (hours)
- Set default session duration
- Delete account (with confirmation)

**Optional:**

- Notification preferences
- Time zone settings
- Theme preference (light/dark mode)

---

## Streak Logic

**Rules:**

- A day counts toward streak if user completes at least 1 deep work session
- Minimum session duration to count: 25 minutes
- Streak breaks if no sessions for 24 hours
- Streak continues across days with sessions

**Display:**

- Current streak prominently on dashboard
- Longest streak in stats
- Calendar heatmap shows all days with sessions

**Motivation:**

- Streaks provide daily accountability
- Visual feedback encourages consistency
- Gamification element without being overwhelming

## User Interface Requirements

### Design Principles:

1. **Minimalist** - Remove distractions, focus on core actions
2. **Clear hierarchy** - Most important info (streak, today) at top
3. **Fast** - No unnecessary clicks, quick load times
4. **Responsive** - Works on desktop, tablet, mobile
5. **Accessible** - Proper contrast, keyboard navigation

### Color Scheme:

- Primary: Focus-oriented (blues, purples)
- Accent: Motivating (orange/red for streaks)
- Neutral: Clean backgrounds (white/light gray)
- Dark mode: Optional for MVP

### Typography:

- Clear, readable fonts
- Large numbers for streak and timer
- Comfortable reading size for text

## Data Requirements

### What Data to Store:

**User Data:**

- Email (unique)
- Password (hashed)
- Full name
- Weekly goal hours
- Default session duration
- Account creation date

**Session Data:**

- Date and time (start/end)
- Duration in minutes
- Title (what they worked on)
- Notes (optional)
- Planned vs ad-hoc flag
- Completed status
- User ID (foreign key)

**Planned Blocks:**

- Day of week (0-6)
- Start time
- Duration
- Title
- Recurring flag
- Active status
- User ID (foreign key)

**Calculated Stats:**

- Current streak
- Longest streak
- Total sessions count
- Total minutes
- Last session date

### Data Security:

- All user data must be private
- Users can only access their own data
- Passwords must be encrypted
- HTTPS for all connections
- No sharing of data between users (for MVP)

## Success Metrics (Post-Launch)

### Primary Metrics:

1. **Daily Active Users (DAU)** - How many users return daily
2. **Retention Rate** - % of users who return after 7 days, 30 days
3. **Average Streak Length** - Indicator of habit formation
4. **Sessions Per User Per Week** - Engagement metric

### Secondary Metrics:

1. **Sign-up conversion rate** - Landing page â†’ registered user
2. **Time to first session** - How quickly users start using the app
3. **Planned vs ad-hoc ratio** - Are users planning ahead?
4. **Average session duration** - Quality of focus time

---

## Out of Scope for MVP

**Explicitly NOT included in first version:**

- Social features (sharing, teams, leaderboards)
- Mobile native apps (web-only for MVP)
- Calendar integrations (Google Calendar, etc.)
- Pomodoro mode with break timers
- Quality ratings for sessions
- Website/app blocking features
- Ambient sounds or music
- AI-powered insights
- Export data functionality
- Third-party integrations
- Collaborative workspaces
- Payment/subscription system

**These can be added in future iterations based on user feedback.**

## Questions to Answer During Development

1. What's the minimum session duration to count toward streak? (Suggested: 25 min)
2. Should we allow sessions to span multiple days? (Suggested: No for MVP)
3. How far in advance can users plan? (Suggested: 4 weeks)
4. What happens to planned blocks that aren't completed? (Suggested: Mark as missed)
5. Can users edit completed sessions? (Suggested: Yes, but log the edit)

---

## Conclusion

This MVP focuses on the core value proposition: helping users build a deep work habit through planning, tracking, and streaks. The key is simplicity and speed to market. We can iterate based on real user feedback rather than building features that might not be needed.

**Timeline:** 7 days from start to launch  
**Budget:** Free tier services for initial launch  
**Goal:** 100 users in first month, 10% retention after 7 days
